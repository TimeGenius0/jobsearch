# 🏗️ Architecture & Data Flow

## File Responsibilities

### 📋 profile.json
**Purpose:** Single source of truth for ALL applicant facts  
**Contents:**
- Personal info (name, email, phone, LinkedIn, GitHub, portfolio)
- Location (city, state, zip, full address)
- Work history (current title, company, years of experience)
- Education (degree, university, graduation year)
- Preferences (work auth, visa needs, remote preference, relocation)
- File paths (resume, cover letter)

**Used by:** Application bot for auto-filling forms

**Example:**
```json
{
  "personalInfo": {
    "firstName": "Bilel",
    "email": "bilel@example.com",
    "linkedin": "https://linkedin.com/in/bilelburaway"
  },
  "work": {
    "currentTitle": "Principal Product Manager",
    "yearsExperience": 15
  }
}
```

---

### 💬 responses.json
**Purpose:** Factual answers to common application questions  
**Contents:**
- Salary expectation
- Start date availability
- Referral source
- EEO questions (optional: veteran status, disability, demographics)

**Used by:** Application bot for dropdown/text fields

**Example:**
```json
{
  "salaryExpectation": "$180,000 - $220,000",
  "availableStartDate": "2 weeks notice",
  "referralSource": "LinkedIn"
}
```

**NOT for:** Persuasive content (that's the cover letter's job)

---

### 📝 cover-letter.docx
**Purpose:** AI-generated persuasive content tailored to each job  
**Contents:**
- Why this company (specific to the job posting)
- Why this role (matches candidate's background)
- Key achievements (selected from resume based on JD)
- Call to action

**Generated by:** `cover_letter.py` (AI analyzes job description)  
**Used by:** Application bot uploads as file attachment

**Flow:**
```
Job URL → AI analysis → Tailored letter → Word template → .docx file → Upload
```

## Separation of Concerns

| File | Type | Changes | Purpose |
|------|------|---------|---------|
| **profile.json** | Static | Rarely | Who you are (facts) |
| **responses.json** | Static | Per-job | Standard answers |
| **cover-letter.docx** | Generated | Every job | Why hire you (persuasion) |
| **resume.pdf** | Static | Occasionally | Experience proof |

---

## Why This Architecture?

### ✅ Benefits

1. **Single source of truth**  
   - Update your email once in profile.json → applies everywhere
   - No duplicate data across files

2. **AI generates what it's good at**  
   - Cover letters = persuasive, tailored content
   - AI reads job description + your resume → custom letter

3. **You control the facts**  
   - Contact info, work history, preferences = you maintain
   - No AI hallucination risk on basic facts

4. **Reusable components**  
   - profile.json = use for 100 applications
   - cover-letter.docx = regenerate per job
   - responses.json = tweak salary range occasionally

5. **Clean separation**  
   - Facts (profile) ≠ Persuasion (cover letter) ≠ Answers (responses)

---

## Example Workflow

**Initial setup (once):**
1. Edit `profile.json` with your real info
2. Edit `responses.json` with standard answers
3. Add `resume.pdf`

**Per job application:**
1. Run: `./apply.sh "job-url" "Company"`
2. AI generates custom `cover-letter.docx`
3. Bot uploads resume + cover letter
4. Bot fills form using profile + responses
5. You review and submit

**Result:** 
- 3 minutes per application (vs 15-20 manually)
- Tailored cover letter every time
- Consistent, accurate data entry

## Maintenance

**Update profile.json when:**
- You change jobs
- You move
- You update LinkedIn/portfolio
- Your phone/email changes

**Update responses.json when:**
- Salary expectations change
- Availability changes
- Targeting different referral sources

**Regenerate cover-letter.docx:**
- Every job application (automated by `apply.sh`)

**Update resume.pdf:**
- When adding new experience
- When tailoring for specific industries
